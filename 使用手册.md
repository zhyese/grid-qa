## 1. 快速上手

| 项 | 值 |
|---|---|
| 前端地址 | `http://localhost:5173`（开发）或部署后的站点地址 |
| 接口文档 | `http://localhost:8001/docs` |
| 健康检查 | `http://localhost:8001/health` |
| 默认账号 | **admin / admin123**（管理员，可用所有功能） |

**登录后**：左侧导航固定可见（可折叠），顶栏显示当前页标题与登录用户/角色，右下角可切换亮/暗主题。

> 首次使用前，知识库需有文档（否则问答会因证据不足而保守拒答）。上传流程见 [§5](#5-知识库管理)。

## 2. 智能问答

最核心页面。像聊天一样用自然语言问电网运维问题。

### 提问
- 底部输入框输入问题（如「主变压器温度异常如何处理」），点「发送」或回车。
- 答案**逐字流式输出**（打字机效果），先返回引用来源，再返回正文。
- 顶部可切换模型（默认 / DeepSeek / 通义千问 / 豆包）与通道（SSE / WS）。

### 答案区会显示
- **引用来源**：点规程段落可定位/复制，每条带文档名。
- **置信度标签**：🟢 高（证据充分）／🟡 有限／🔴 不足（系统保守拒答，零幻觉）。
- **🔗 图谱 N**：答案融合了知识图谱的因果三元组条数。
- **⚠ 高风险提示**：答案含停电/接地/倒闸等高风险操作时标红，务必核对安规再执行。
- **相关追问**：答完自动推 3 个相关问题，点击即问。
- **操作栏**：重新生成 / 复制 / 导出 Word（生成 .docx 运维报告，可打印归档）。
- **反馈**：👍/👎。点 👎 会把该 case 沉淀到管理员反馈台，便于纠错回流。

### 对话控制三件套（高频操作）
- **⏹ 停止生成**：流式输出过程中，发送钮变成「停止」，点击立即中断；已收到的部分答案保留并标注「[已中断]」。
- **🔄 重新生成**：对任一回答点「重新生成」，以**原问题**重新作答（跳过缓存，结果作为新一轮追加，原答案保留，方便对比）。
- **✏️ 编辑重提**：对任一提问点「编辑」，就地修改文本后「重提」——以改后的问题作为**新一轮追问**发送，**原问答历史保留**（不改写历史，保证多轮上下文连续）。

### 对话管理（左侧栏）
- 「+ 新建对话」开新会话；支持搜索、重命名、删除。
- 多轮追问会自动带上近几轮上下文（指代消解，如「它的处置步骤呢」）。

## 3. 故障诊断

进入「🩺 故障诊断」页，顶部 4 个标签页切换：

### 3.1 🩺 故障诊断推理
- 输入**故障症状**（如「1号主变上层油温 88℃ 持续上升，负载 70%」），点「开始诊断」。
- 系统自动多查询分解 → 并行检索 + 知识图谱因果链 → 给出：
  - 排查方向（多查询分解出的几个方向）
  - **可能原因**按可能性排序（高/中/低），每个带「依据」和「处置措施」
  - ⚠ 风险点提示 + 规程依据来源

### 3.2 📚 相似案例
- 输入当前故障/症状，在**故障案例库**（docType=故障案例 的文档）里找历史相似案例。
- 返回案例文档名、相关度、内容片段。"历史上类似故障怎么处理的"。

### 3.3 📝 两票生成
- 输入**操作任务**（如「1号主变压器由运行转检修」），生成结构化操作票草案：涉及设备、操作步骤（按规程顺序）、安全措施、风险点、备注。
- ⚠ 这是**辅助草案**，必须由持票人核对调度指令与安规后才能执行。

### 3.4 🔍 两票审核（新功能）
- 对**已填好的票据**做合规审核。详见 [§4](#4-两票智能审核专题)。

## 4. 两票智能审核专题

> 电网安监的核心是"审"。把已填的操作票/工作票全文粘贴进来，系统用**规则引擎 + LLM 双层**做合规审核，输出结构化报告，帮人工审核员快速定位问题。
> **权限：仅管理员（admin）可用。** 普通用户点「审核」会提示需要管理员权限。

### 4.1 怎么用
1. 在「🩺 故障诊断 → 🔍 两票审核」tab。
2. 选择票据类型：**操作票 / 工作票**。
3. （可选）切换审核用模型。
4. 把**票据全文**粘进大文本框，点「开始审核」。
5. 约 5–10 秒返回审核报告。

### 4.2 票据文本建议格式
为获得最准的解析，粘贴的票据建议包含以下字段（带冒号标签、步骤用编号），缺项也能审（会报"必填项缺失"）：

```
操作任务：1号主变由运行转检修
调度指令号：DD-2026-001
操作人：张三
操作步骤：
1. 断开1号主变10kV侧断路器
2. 验电
3. 挂地线
安全措施：
- 戴绝缘手套
危险点：
- 触电
```

- 工作票用「工作任务 / 工作负责人」也能识别。
- 步骤支持 `1.` `1、` `1)` `①` `-` 等多种编号格式。

### 4.3 审核报告怎么读
报告结构：

| 字段 | 含义 |
|---|---|
| **overall** | 总体结论：`pass`（✓ 合规）/ `warn`（⚠ 需修改）/ `fail`（✗ 不合规） |
| **score** | 0–100 得分。`≥85` pass、`60–84` warn、`<60` fail |
| **ticketType** | 票据类型 |
| **items[]** | 问题清单，每条含 layer / ruleId / severity / msg / suggestion |
| **latencyMs** | 审核耗时（毫秒） |

每条问题的严重度徽章：
- 🔴 **critical（严）**：严重违安规，必须改（如「挂地线在验电之前」「缺操作任务」）
- 🟡 **major（重）**：重要缺项/表述（如缺操作人、含禁用术语「约时停送电」）
- ⚪ **minor（轻）**：轻微微调（如调度指令号格式不规范）

每条还带**修正建议**（suggestion），可直接照改。

### 4.4 审核规则（双层）
**层 1 · 规则引擎（确定性、零成本、可配）：**

| 规则 | ruleId 前缀 | 说明 |
|---|---|---|
| 必填项缺失 | `REQ_*` | 任务/调度指令号/操作人 缺失 |
| 操作顺序违安措 | `SEQ_*` | 如「挂地线」出现在「验电」之前、「接地」在「验电」之前 |
| 高风险无危险点 | `DANGER_001` | 含停电/接地/倒闸等高风险操作却没列危险点 |
| 调度指令号格式 | `DISP_001` | 编号不符合规范格式（字母数字+连字符，≥4 位） |
| 禁用术语 | `BLOCK_001` | 含「约时停送电」「口头指令」等安规禁用表述 |

**层 2 · LLM 语义审核：** 查规则引擎查不到的语义类问题——安措是否覆盖所有危险点、术语是否规范、步骤是否可执行、有无遗漏关键步骤。输出以 `LLM_*` 开头的条目（layer=llm）。

> **降级**：若 LLM 临时不可用（超时/欠费），系统自动降级为**只返回规则层结果**（仍可用），并在监控里记一次降级。

### 4.5 打分规则
单项扣分：critical=35、major=15、minor=5。`得分 = 100 − Σ扣分`（不低于 0），再映射到 pass/warn/fail。所以：一条严重违安规就足以让票据落到 warn，两条以上严重问题通常即 fail。

### 4.6 规则可配置（管理员）
规则定义在 `backend/data/ticket_rules.json`，结构与内置默认规则一致。改完**重启后端**生效；文件缺失或格式错时自动回落内置默认规则（规则层始终可用）。

## 5. 知识库管理

进入「📄 知识库」页。这是问答/诊断能否答得准的地基。

### 流程：上传 → 解析 → 向量化
1. **上传**：拖拽或选择文件（PDF / Word / Excel / TXT / 图片，单文件 ≤100M，批量 ≤5 个），带进度条。
2. **解析**：对已上传文档点「解析」——结构感知分块（表格整体保留、父子两层）、OCR（扫描件/图片）、术语归一化、设备自动打标。
3. **向量化**：解析完后点「向量化」——按文档大小自动路由（大走云 Embedding、小走本地 bge），写入向量库。
4. **完成后**该文档即参与问答/诊断检索。

### 其他操作
- **列表**：分页 + 关键字搜索；按类型/状态筛选；批量勾选。
- **预览**：点文档名弹窗看原文（PDF/图片/文本）。
- **统计**：页面顶部卡片显示文档数/分块数/向量数及分布。
- **删除**：联动清理 MinIO 原文 + Milvus 双向量库 + MySQL + Neo4j，不留残渣。
- **版本**：同名文档换版会自动归档旧版，支持查历史版本与回滚（管理员）。

> 提示：上传后**必须依次完成「解析」和「向量化」**，否则检索不到。状态列会显示当前进度。

## 6. 统计看板

「📊 统计看板」——echarts 仪表盘，一眼掌握系统运行状况：
- 知识库规模（文档/分块/向量）、状态与类型分布。
- 故障反馈趋势（dislike 率、坏 case 设备聚类、高频问题、平均幻觉率）——反哺知识库优化。

## 7. 知识图谱

「🧠 知识图谱」——基于 Neo4j 的设备-故障-处置多跳推理，三个 tab：
- **关系图谱**：输入实体（如「主变压器」），查邻居关系（力导向图）。Neo4j 不可用时回退 MySQL。
- **多跳影响链**：查因果传播链（设备 → 故障 → 处置），看一个设备异常会连带影响什么。
- **枢纽实体**：按出度排行找核心设备（连接关系最多的）。

问答与诊断会自动融合图谱上下文（答案带 🔗图谱N 标签）。

## 8. 检索调试（管理员）

「🔬 检索调试」（仅 admin 可见）——排障调参神器。
- 输入 query，看到**检索全链路 trace**：query 改写 / HyDE / 多查询分解 / 双路 dense + BM25 / RRF 融合 / rerank / 元数据过滤 / MMR，每一步中间结果都透出。
- 每条命中带**分数归因**（dense 分 / bm25 分 / rrf 分 / rerank 分 / final），「为什么没命中某文档」一目了然。
- 展示当前运行时配置（Milvus HNSW `ef`、模型 `temperature`）。

---

## 9. 系统管理（管理员）

「⚙️ 系统管理」（仅 admin）——多个 tab：
- **反馈**：坏 case 看板（judge 分/理由），「标为 golden」一键回流回归集，永久覆盖该 case。
- **操作日志**：全量操作审计（按时间/用户过滤）。
- **告警**：Grafana 告警回调落地 here（组件下线/降级激增/幻觉率/安全命中），不依赖钉钉/企微。
- **配置**：Milvus 索引参数、模型 temperature（**运行时即时生效**，无需重启）；LLM/Embedding provider 主动探测（抓欠费/key 失效）；BM25 全量重建按钮。

## 11. 常见问题排查

**Q：问答页显示「流式中断：流式请求失败: 500」/ 页面空白卡住？**
A：八成是后端没起或正在重启。先：
1. 浏览器 **Ctrl+F5 硬刷新**（清掉旧的 SPA 缓存状态）。
2. 访问 `http://localhost:8001/health`，看是否返回 `status=healthy`。不可达 → 后端进程挂了，重启后端。
3. 若 health 正常仍 500 → 看 `data/logs/app.log` 末尾的异常栈定位。

**Q：两票审核提示「审核失败（需管理员权限）」？**
A：该接口仅 admin。用 admin 账号登录即可（或让管理员给你提权）。

**Q：两票审核第一次很慢（几十秒）？**
A：首次审核会触发本地 bge 模型加载 + LLM 首次连接建立，属一次性冷启动；后续审核约 5–10 秒。前端超时设置 60 秒，正常不会超时。

**Q：答案总是「拒答（🔴不足）」？**
A：说明检索证据不足（CRAG 自纠错保守拒答，防幻觉）。检查：知识库里是否上传并**解析+向量化**了相关规程文档；文档 docType 是否正确。

**Q：上传文档后检索不到？**
A：确认该文档状态已是「已解析 + 已向量化」（知识库页状态列）。只上传不解析/向量化，不会进检索。

**Q：切换 LLM 模型？**
A：问答/诊断/审核的输入框旁有模型下拉，按请求切换；或管理员改 `.env` 的 `LLM_PROVIDER` 改默认（重启生效）。

**Q：阿里百炼欠费致 rerank/embed 失败？**
A：系统会自动降级（rerank 失败回退 RRF），监控里可见降级计数；恢复需充值百炼。
