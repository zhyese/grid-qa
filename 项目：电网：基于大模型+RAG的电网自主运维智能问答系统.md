# 基于大模型+RAG的电网自主运维智能问答系统
**项目类型**：电力行业AI知识中台
**项目角色**：算法&后端核心研发

## 一、项目背景
电网运维涉及**变电、配电、输电**三大场景，包含设备说明书、故障案例、检修规程、作业指导书、反事故措施、缺陷库等海量非结构化专业文档。
传统模式依赖**关键词检索+人工翻阅**，存在三大痛点：
1. 专业术语多、歧义强，关键词检索召回率低、相关性差；
2. 故障处置需跨文档查找，响应慢、易出错；
3. 知识沉淀依赖老员工，新人上手周期长。

为此自研**RAG检索增强生成问答系统**，实现**自然语言提问 → 精准检索 → 可信答案生成 → 标准化输出**，为一线运维提供可直接落地的故障处理方案。

## 二、核心技术栈（全链路）
- **RAG框架**：LangChain + LlamaIndex（双框架兼容，增强鲁棒性）
- **大模型**：Qwen / Baichuan / ChatGLM（支持本地私有化部署）
- **向量模型**：bge-large-zh-v1.5（电网领域微调后嵌入模型）
- **向量数据库**：Milvus 2.3（高性能分布式向量检索）
- **文档处理**：PDFMiner、python-docx、文本分块、语义增强
- **检索引擎**：向量相似度检索 + BM25稀疏检索 + 交叉编码器重排
- **后端服务**：FastAPI、Uvicorn
- **部署环境**：Docker Compose、Linux、CUDA 11.7
- **存储**：MySQL（元数据）+ MinIO（源文档）+ Milvus（向量）

## 三、责任描述
1. **知识文档处理模块（核心研发）**
   - 实现**专业文档结构化解析**，支持PDF/Word/扫描件文本提取、目录识别、表格抽取；
   - 设计**自适应语义分块算法**，按章节/段落/语义边界切分，保证块内语义独立；
   - 针对电网领域特点做**语义增强**：补充设备编号、电压等级、故障类型、术语归一化；
   - 构建**领域词表与停用词表**，优化专业术语表达一致性。

2. **向量生成与存储模块（全链路实现）**
   - 选用并**微调领域嵌入模型**，提升电网专业文本向量表征能力；
   - 搭建批处理向量化 pipeline，完成百万级文本块高维向量（1024维）生成；
   - 构建Milvus向量库，建立**IVF_FLAT索引**，配置余弦相似度检索策略；
   - 实现向量库增量更新、文档版本管理、元数据关联存储。

3. **查询处理与检索模块（算法核心）**
   - 对用户查询进行**意图识别、术语纠错、专业词扩展**；
   - 构建**混合检索机制**：稠密向量检索 + BM25全文检索，提高召回率；
   - 基于**交叉编码器（Cross-Encoder）** 对候选文本块做相关性重排序；
   - 实现**上下文窗口过滤**，按相关性、时效性、权威性筛选Top-K片段。

4. **答案生成与返回模块（工程化落地）**
   - 设计**电网领域动态Prompt模板**，包含约束条件、输出格式、专业要求；
   - 构建**上下文拼接策略**，将重排序后的文本块结构化输入大模型；
   - 实现答案**后处理**：去幻觉、格式标准化、引用标注、风险提示；
   - 封装标准化API，提供问答、检索、溯源、反馈一体化接口。

5. **系统架构与联调**
   - 负责四大模块**数据接口定义与通信**，实现文档→向量→检索→生成全链路打通；
   - 优化检索速度、模型推理速度，满足高并发在线服务；
   - 构建效果评估体系：**召回率、精确率、命中率、答案准确率**自动化评测。

## 四、项目业绩（可量化+技术指标）
1. **检索效果**：专业知识召回率 ≥ **92%**，检索精确率 ≥ **88%**，较关键词检索提升**120%+**；
2. **问答效果**：电网运维场景答案准确率达 **91%**， hallucination（幻觉率）控制在 **5%以内**；
3. **性能指标**：单轮问答响应 **≤ 2.5s**，支持 **100+并发**，满足现场实时使用；
4. **效率提升**：运维故障查询时间从**10~15分钟**降至**10秒内**，效率提升 **90%+**；
5. **工程价值**：基于开源RAG框架快速落地，**研发周期缩短60%**，支持一键切换大模型；
6. **业务价值**：覆盖26类设备、800+故障模式、10万+专业知识条目，已在区域电网正式上线。
