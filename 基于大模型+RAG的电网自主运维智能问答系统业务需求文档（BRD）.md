# 基于大模型\+RAG的电网自主运维智能问答系统业务需求文档（BRD）

# 1\. 文档前言

## 1\.1 文档目的

本文档旨在明确“基于大模型\+RAG的电网自主运维智能问答系统”（以下简称“本系统”）的业务需求、功能边界、用户诉求、非功能要求及验收标准，为系统设计、开发、测试、部署及运维提供统一的业务依据，确保系统落地后能够满足电网运维场景的实际业务需求，解决传统运维知识检索与应用的痛点，提升运维效率与专业化水平。

## 1\.2 文档范围

本文档覆盖本系统的全业务流程、核心功能模块、用户群体、业务目标、数据需求、非功能需求、约束条件及验收标准，适用于系统研发团队、测试团队、运维团队、业务对接团队及相关决策人员，作为系统建设的核心指导文档。

## 1\.3 术语与定义

- RAG（Retrieval\-Augmented Generation）：检索增强生成，结合检索技术与大语言模型，生成基于真实知识的可信答案，减少模型幻觉。

- 电网自主运维：电网运维人员通过系统自主获取设备运维、故障处置、规程查询等相关知识，实现故障快速定位与解决，降低对资深人员的依赖。

- 非结构化专业文档：电网运维场景中无固定格式的文档，包括设备说明书、故障案例、检修规程、作业指导书、反事故措施、缺陷库等。

- 向量嵌入：通过嵌入模型将文本转化为高维向量，用于语义相似度计算与检索。

- 混合检索：结合稠密向量检索（语义匹配）与BM25稀疏检索（关键词匹配），提升检索召回率与相关性。

- 幻觉率（Hallucination）：大语言模型生成的答案中，与真实知识不符、无依据的内容占比。

# 2\. 项目概述

## 2\.1 项目背景

电网运维是保障电网安全稳定运行的核心环节，涉及变电、配电、输电三大核心场景，积累了海量非结构化专业文档，包括设备说明书、故障案例、检修规程、作业指导书、反事故措施、缺陷库等，这些文档是运维人员开展工作的核心知识依据。

当前电网运维知识应用采用“关键词检索\+人工翻阅”的传统模式，受限于电网专业术语多、歧义性强、知识分散等特点，存在以下核心痛点：

1. 检索效果差：关键词检索无法精准匹配语义，导致召回率低、相关性差，运维人员需花费大量时间筛选无效信息。

2. 响应效率低：故障处置时需跨多个文档查找相关知识，流程繁琐、响应缓慢，易因知识获取不及时导致操作失误。

3. 知识沉淀难：运维知识高度依赖资深员工的经验积累，新人上手周期长，知识传承存在断层风险。

4. 开发成本高：传统定制化问答系统研发周期长、扩展性差，无法快速适配电网运维知识的更新与业务场景的变化。

为解决上述痛点，依托大模型与RAG开源框架，自研电网自主运维智能问答系统，实现“自然语言提问→精准检索→可信答案生成→标准化输出”的全流程自动化，为一线运维人员提供可直接落地的故障处理方案，同时降低开发成本、增强系统扩展性。

## 2\.2 项目目标

### 2\.2\.1 业务目标

1. 提升运维效率：将运维故障查询时间从传统的10\~15分钟缩短至10秒内，效率提升90%以上，降低一线运维人员的工作负担。

2. 提高问答准确性：电网运维场景答案准确率达到91%以上，幻觉率控制在5%以内，确保答案可落地、可追溯。

3. 优化知识应用：实现运维知识的结构化沉淀与高效检索，打破知识壁垒，缩短新人上手周期，实现知识标准化传承。

4. 降低研发与运维成本：基于开源RAG框架快速落地，缩短60%的研发周期，支持一键切换大模型，降低系统迭代与维护成本。

5. 实现业务落地：覆盖26类电网设备、800\+故障模式、10万\+专业知识条目，成功在区域电网正式上线并稳定运行。

### 2\.2\.2 技术目标

1. 检索效果：专业知识检索召回率≥92%，检索精确率≥88%，较传统关键词检索提升120%以上。

2. 性能指标：单轮问答响应时间≤2\.5s，支持100\+并发请求，满足现场实时运维需求。

3. 系统鲁棒性：支持双RAG框架（LangChain \+ LlamaIndex）兼容，多大模型（Qwen/Baichuan/ChatGLM）切换，具备故障自动恢复能力。

4. 可扩展性：支持文档增量更新、知识体系扩展，适配电网运维业务的动态变化，可快速新增设备类型、故障模式及知识条目。

## 2\.3 项目范围

### 2\.3\.1 核心业务范围

本系统聚焦电网运维场景，核心覆盖变电、配电、输电三大场景的知识检索与问答服务，具体包括：

- 运维知识检索：支持自然语言查询设备说明书、故障案例、检修规程、作业指导书等专业文档相关内容。

- 故障处置问答：针对电网设备故障，提供精准的故障原因分析、处置步骤、注意事项等可落地方案。

- 知识管理：支持运维知识文档的上传、解析、结构化处理、增量更新及版本管理。

- 系统管理：支持用户权限分配、操作日志记录、系统性能监控、数据备份与恢复等。

### 2\.3\.2 排除范围

- 不涉及电网设备实时监控数据的采集与分析（仅依托现有文档知识，不对接实时监控系统）。

- 不涉及运维作业的在线执行与闭环管理（仅提供知识支持，不参与实际作业流程管控）。

- 不涉及非电网运维领域的知识问答（如电网规划、调度等场景暂不覆盖）。

# 4\. 核心功能需求

本系统的核心功能围绕四大模块展开，分别为知识文档处理模块、向量生成与存储模块、查询处理与检索模块、答案生成与返回模块，各模块功能相互衔接，实现全链路的问答服务，同时配套系统管理功能，确保系统正常运行。

## 4\.1 知识文档处理模块

核心功能：实现电网运维专业文档的结构化解析、自适应分块、语义增强及词表优化，为后续向量生成与检索提供高质量的文本数据。

1. 文档解析功能
        

    - 支持多种文档格式解析，包括PDF、Word、扫描件（需支持OCR识别），可提取文档中的文本、表格、目录等内容。

    - 实现目录自动识别与关联，确保分块与文档原有结构一致，便于后续溯源。

    - 支持批量上传文档，可设置上传权限（仅管理人员可上传），上传后自动触发解析流程。

    - 解析完成后，生成解析报告，标注解析成功/失败状态，失败时提示具体原因（如文档损坏、格式不支持）。

2. 自适应语义分块功能
       

    - 采用自适应分块算法，可根据文档的章节、段落、语义边界自动切分文本，确保每个文本块语义独立，避免语义断裂。

    - 支持分块参数自定义（如块大小、重叠率），可根据文档类型（如设备说明书、故障案例）调整分块策略。

    - 分块后，为每个文本块添加元数据（如文档名称、章节、页码、设备类型、故障类型），便于后续检索与溯源。

3. 语义增强功能
       

    - 结合电网领域特点，对文本块进行语义增强，补充设备编号、电压等级、故障类型、专业术语等关键信息，提升向量表征的准确性。

    - 实现专业术语归一化，将同义术语（如“SF6断路器”与“六氟化硫断路器”）统一表述，避免因术语差异影响检索效果。

    - 支持语义增强规则自定义，可根据业务需求新增增强维度（如新增作业场景、安全等级等）。

4. 词表管理功能
        

    - 构建电网领域词表与停用词表，词表包含设备名称、故障类型、专业术语等核心词汇，停用词表过滤无意义词汇（如“的、地、得”）。

    - 支持词表的新增、编辑、删除，管理人员可根据业务变化更新词表，优化文本处理效果。

    - 实现术语纠错功能，基于领域词表，对文档中的术语错误进行提示与修正（如“变压气”修正为“变压器”）。

5. 文档版本管理功能
        

    - 支持文档版本记录，每次更新文档后，保留历史版本，可查看版本差异、回滚至历史版本。

    - 为每个版本添加更新记录（更新人、更新时间、更新内容），便于追溯管理。

## 4\.2 向量生成与存储模块

核心功能：将处理后的文本块转化为高维向量，存储到向量数据库中，实现向量的高效管理与增量更新，为检索模块提供数据支撑。

1. 向量生成功能
        

    - 选用bge\-large\-zh\-v1\.5嵌入模型，并结合电网领域数据进行微调，提升专业文本的向量表征能力。

    - 搭建批处理向量化流水线，支持百万级文本块的批量向量生成，生成1024维高维向量。

    - 向量生成过程中，关联文本块的元数据，确保向量与原始文本、文档信息一一对应。

    - 支持向量生成状态监控，可查看生成进度、成功/失败数量，失败时自动重试。

2. 向量存储功能

    - 采用Milvus 2\.3向量数据库，构建IVF\_FLAT索引，配置余弦相似度检索策略，提升向量检索速度与准确性。

    - 实现向量与元数据的关联存储，将向量、文本块、文档信息、分块信息等关联保存，便于检索后溯源。

    - 支持向量数据库的分片与扩容，满足海量向量数据的存储需求。

3. 向量更新与维护功能
        

    - 支持向量库增量更新，当文档新增、更新时，自动生成对应的向量并添加到向量数据库，无需重新生成全部向量。

    - 实现向量删除功能，当文档删除时，自动删除对应的向量及相关元数据，避免无效数据占用存储空间。

    - 支持向量数据库健康检查，监控数据库运行状态，及时发现并处理存储异常。

## 4\.3 查询处理与检索模块

核心功能：处理用户输入的自然语言查询，转化为查询向量，通过混合检索机制检索相关文本块，经重排序后输出候选文本集合，为答案生成提供依据。

1. 查询处理功能
        

    - 支持用户自然语言输入，无需严格按照关键词格式提问，可识别口语化、专业化的查询语句（如“变压器跳闸了怎么办”“线路接地故障排查步骤”）。

    - 实现查询意图识别，区分用户的查询类型（故障处置、知识查询、学习求助等），针对性优化检索策略。

    - 提供术语纠错与扩展功能，基于领域词表，对查询中的术语错误进行修正，对专业词进行同义词扩展（如查询“SF6断路器”时，同时检索“六氟化硫断路器”相关内容）。

    - 支持查询历史记录，用户可查看自己的查询历史，重新发起查询，节省操作时间。

2. 混合检索功能


    - 采用“稠密向量检索\+BM25稀疏检索”的混合检索机制，稠密向量检索负责语义匹配，BM25检索负责关键词匹配，提升检索召回率。

    - 将用户查询语句通过嵌入模型转化为查询向量，利用余弦相似度算法从向量数据库中检索相关文本块。

    - 支持检索参数自定义（如Top\-K值、相似度阈值），可根据业务需求调整检索精度与召回率。

3. 相关性重排序功能
        

    - 基于交叉编码器（Cross\-Encoder）对检索到的候选文本块进行相关性重排序，优先输出与用户查询最相关的文本块。

    - 重排序过程中，结合文本块的相关性、时效性（文档更新时间）、权威性（文档类型优先级，如规程\&gt;案例\&gt;说明书）进行综合评分。

    - 实现上下文窗口过滤，筛选出Top\-K（可配置）个最优文本块，作为答案生成的输入，避免冗余信息影响答案质量。

## 4\.4 答案生成与返回模块

核心功能：基于重排序后的候选文本块，结合动态提示模板生成大模型输入，生成可信答案并进行后处理，最终以标准化格式返回给用户，同时提供答案溯源功能。

1. 提示模板设计功能
        

    - 设计电网领域动态Prompt模板，模板包含约束条件（禁止生成无依据内容）、输出格式（结构化、分点表述）、专业要求（符合电网运维规范）。

    - 支持模板自定义，管理人员可根据业务场景（如故障处置、知识查询）调整模板内容，优化答案生成效果。

    - 模板可根据用户查询意图自动匹配，确保生成的答案贴合用户需求。

2. 答案生成功能
        

    - 构建上下文拼接策略，将重排序后的文本块按相关性优先级结构化拼接，结合动态Prompt模板，生成大语言模型的输入。

    - 支持多大模型切换（Qwen/Baichuan/ChatGLM），可根据系统性能与问答效果选择合适的大模型，支持本地私有化部署，保障数据安全。

    - 生成答案时，确保答案基于候选文本块，减少模型幻觉，同时保留专业术语的准确性与规范性。

3. 答案后处理功能


    - 去幻觉处理：对生成的答案进行校验，过滤无依据、与候选文本块不符的内容，将幻觉率控制在5%以内。

    - 格式标准化：将答案按固定格式整理（如分点表述、标注关键信息），便于用户快速阅读与执行，故障处置类答案需包含“故障原因、处置步骤、注意事项”三部分。

    - 引用标注：在答案中标注引用来源（如“参考《XX检修规程》第X条”“参考XX故障案例”），用户可点击来源查看原始文本块。

    - 风险提示：对高风险操作（如带电作业、设备检修）添加安全风险提示，提醒用户严格按照规程操作。

4. 答案返回与交互功能
        

    - 将后处理后的答案以清晰、直观的格式返回给用户，支持文本复制、分享功能。

    - 提供答案反馈功能，用户可对答案的准确性、实用性进行评分（如“准确”“基本准确”“不准确”），并填写反馈意见，用于系统优化。

    - 支持答案溯源，用户可点击答案中的引用来源，查看对应的原始文本块及关联文档，确保答案的可信度。

5. API封装功能
       

    - 封装标准化API接口，提供问答、检索、溯源、反馈等一体化服务，支持与其他电网运维系统对接。

    - API接口支持高并发请求，提供详细的接口文档，便于第三方系统集成。

## 4\.5 系统管理模块

核心功能：保障系统稳定运行，实现用户、权限、数据、日志的全流程管理，支持系统性能监控与优化。

1. 用户与权限管理


    - 支持用户账号的新增、编辑、禁用、删除，用户可修改个人密码、个人信息。

    - 基于角色的权限控制（RBAC），创建三类角色（一线运维人员、运维管理人员、系统管理员），分配不同的操作权限，确保数据安全与操作规范。

    - 支持权限批量分配与修改，便于管理员高效管理用户权限。

2. 日志管理
        

    - 记录系统所有操作日志，包括用户登录、查询、文档上传、系统设置等，日志包含操作人、操作时间、操作内容、操作结果。

    - 支持日志查询、筛选、导出，可按操作人、操作时间、操作类型等条件查询日志，便于故障排查与责任追溯。

    - 日志自动归档，设置归档周期，避免日志过多占用存储空间。

3. 数据管理
        

    - 支持系统数据（文档数据、向量数据、元数据、日志数据）的备份与恢复，可设置自动备份周期（如每日备份），也可手动触发备份。

    - 支持数据清理功能，可清理过期数据、无效数据，释放存储空间。

    - 数据存储采用MySQL（元数据）\+ MinIO（源文档）\+ Milvus（向量）的组合方式，确保数据安全与高效访问。

4. 系统监控与优化

    - 实时监控系统性能，包括CPU、内存、磁盘占用、并发量、响应时间等指标，设置阈值告警，当指标超出阈值时，及时提醒管理员。

    - 支持系统参数配置，可调整检索参数、向量生成参数、模型参数等，优化系统性能与问答效果。

    - 支持大模型、嵌入模型的切换与升级，可一键切换不同模型，无需修改系统代码。

# 5\. 非功能需求

## 5\.1 性能需求

1. 响应时间：单轮问答响应时间≤2\.5s，文档解析（单篇500页以内）≤30s，批量向量化（10万条文本块）≤2小时。

1. 系统可用性：全年可用性≥99\.9%，单次故障恢复时间≤30分钟，支持故障自动恢复（如向量生成失败自动重试、数据库异常自动切换）。

2. 数据可靠性：数据备份采用异地备份策略，备份成功率100%，数据恢复后无丢失、无损坏，恢复时间≤1小时。

3. 鲁棒性：支持双RAG框架兼容，当一个框架出现异常时，可自动切换至另一个框架，不影响系统正常运行；支持多模型冗余，避免单一模型故障导致系统不可用。

4. 容错性：对用户输入的无效查询（如无意义文本、非运维相关查询），给出友好提示；对文档解析、向量生成等过程中的异常，记录日志并给出处理建议，不导致系统崩溃。

## 5\.3 可扩展性需求

1. 功能扩展：支持新增功能模块（如知识图谱集成、故障预测建议），可通过插件化方式扩展，无需修改核心代码。

## 5\.4 安全性需求

2. 权限安全：严格的权限控制，不同角色仅能访问对应权限的功能与数据，防止越权操作；用户密码采用加密存储，不明文传输。

3. 接口安全：API接口支持身份认证（如Token认证），防止非法调用；对接口请求进行频率限制，避免恶意攻击。

4. 合规性：系统数据处理符合电力行业数据安全规范，不泄露电网核心机密信息，满足数据隐私保护要求。

# 6\. 数据需求

|---|---|---|
|专业文档数据|设备说明书、故障案例、检修规程、作业指导书、反事故措施、缺陷库等非结构化文档|电网运维部门提供、管理人员上传|

1. 存储介质：采用MySQL存储元数据（用户数据、日志数据、文档信息等），MinIO存储源文档，Milvus存储向量数据，确保数据存储的高效性与安全性。

2. 存储容量：支持至少10万\+专业知识条目、百万级文本块、千万级向量数据的存储，预留50%的存储空间用于后续扩展。

4. 数据生命周期：文档数据、向量数据永久保存（除非手动删除），日志数据保存6个月，过期自动归档或清理。

1. 准确性：专业文档数据、术语数据需准确无误，符合电网运维行业规范；文本块解析、向量生成无错误，确保检索与问答的准确性。

4. 时效性：运维知识文档需及时更新，确保知识的时效性；向量数据需随文档更新同步增量更新，避免无效数据。
